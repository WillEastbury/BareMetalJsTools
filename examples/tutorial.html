Build a Dashboard

A step-by-step tutorial showing how every BareMetalJsTools module works together to build a complete, interactive operations dashboard — with zero dependencies and no build step.

What you'll use: BareMetalStyles.css (layout + components), BareMetalCharts.js (data viz), BareMetalGraph.js (architecture diagram), and BareMetalBind.js (reactivity). That's it — 4 files, ~90KB total.

Every code snippet below is live — the rendered result appears right after it. Scroll down to follow the tutorial, or use the sidebar to jump to a step.

1 Page Shell

Start with the letterbox layout — a fixed header and footer with scrollable content between them. Add a sidebar for navigation.

<link rel="stylesheet" href="BareMetalStyles.css"> <div class="page-wrapper"> <header class="site-header site-header-dark"> <a class="brand">🔩 My Dashboard</a> </header> <div class="page-body"> <aside class="site-sidebar sidebar-left">...</aside> <main class="main-content p-4">...</main> </div> <footer class="site-footer site-footer-dark">...</footer> </div>

7 Toast Notifications

Toasts are pure CSS — no JavaScript needed. They slide in, and auto-dismiss toasts fade themselves out after a set duration using CSS animations. Add a .toast-progress bar for a visual countdown.

<!-- Container positions toasts on screen --> <div class="toast-container toast-container-top-right"> <!-- Manual toast (stays until removed) --> <div class="toast toast-success"> <div class="toast-header"> <span class="toast-title">Saved</span> <span class="toast-time">just now</span> </div> <div class="toast-body">Changes saved.</div> </div> <!-- Auto-dismiss after 5s --> <div class="toast toast-auto toast-info"> <div class="toast-body">This will disappear...</div> <div class="toast-progress"></div> </div> </div>
Result — click the buttons to fire toasts

Zero JS needed for styling! The .toast-auto class uses CSS @keyframes to auto-dismiss after 5s (customise with .toast-3s, .toast-8s, .toast-10s). The only JS here is creating the DOM element when you click the button — the animation and removal is pure CSS.

Reactive Toast Binding

Even better — bind a toast container to an array with m-toast. Just push objects and toasts appear automatically:

<!-- Bind toast container to state array --> <div class="toast-container toast-container-top-right" m-toast="notifications"></div> const { state } = BareMetalBind.reactive({ notifications: [] }); // Push an object → toast pops up and auto-dismisses state.notifications.push({ type: 'success', // success | danger | warning | info | dark title: 'Deployed', message: 'v2.1.0 is live!', duration: '5s' // 3s | 5s | 8s | 10s });
Result — click to fire reactive toasts

8 Architecture Diagram

10 Reactive Binding

Use BareMetalBind to create reactive state that auto-updates the DOM. Bind inputs with m-value, text with m-text, visibility with m-if.

<script src="BareMetalBind.js"></script> const state = BareMetalBind.create({ name: '', count: 0 }); BareMetalBind.bind(document.getElementById('app'), state); // Now any change to state.name or state.count auto-updates the DOM state.count++; // UI updates instantly
